# 新增功能描述文档

## 功能概述
在原有支付收银台工程基础上，新增了**支付设置页面**功能，为用户提供个性化的支付配置选项。

## 新增文件列表

### 1. 数据模型文件
**文件路径**: `entry/src/main/ets/data/SettingsData.ets`

**功能描述**:
- 定义支付确认方式枚举（`ConfirmMethod`）：支付密码、指纹识别、人脸识别、无需确认
- 定义支付设置数据模型（`PaymentSettings`）
- 提供可用支付方式列表
- 创建全局设置实例

**主要数据结构**:
```typescript
export class PaymentSettings {
  defaultPaymentMethod: string = '微信支付';  // 默认支付方式
  paymentReminder: boolean = true;            // 支付提醒开关
  smallAmountNoPassword: boolean = false;     // 小额免密支付
  smallAmountLimit: number = 100;             // 小额免密限额
  confirmMethod: ConfirmMethod = ConfirmMethod.PASSWORD;  // 支付确认方式
  autoSaveCard: boolean = true;               // 自动保存银行卡
  showTransactionDetails: boolean = true;     // 显示交易详情
}
```

### 2. 页面文件
**文件路径**: `entry/src/main/ets/pages/Settings.ets`

**功能描述**:
- 支付设置主页面，提供完整的设置界面
- 使用HarmonyOS原生UI组件构建
- 支持底部弹窗选择器（bindSheet）

**主要功能模块**:

#### 2.1 支付方式设置
- 默认支付方式选择
- 支持5种支付方式：微信支付、支付宝、云闪付、银行卡等
- 底部弹窗选择器交互

#### 2.2 安全设置
- 支付确认方式设置（支付密码/指纹/人脸/无确认）
- 小额免密支付开关（可设置限额）
- 提供安全保障选项

#### 2.3 通知设置
- 支付提醒开关
- 支付成功/失败时发送通知

#### 2.4 其他设置
- 自动保存银行卡信息
- 显示交易详情开关

#### 2.5 保存功能
- 保存设置按钮
- 设置持久化到全局变量
- 保存成功提示对话框

## 技术实现要点

### 1. UI组件使用
- 使用 `Toggle` 组件实现开关功能
- 使用 `bindSheet` 实现底部弹窗选择器
- 使用 `AlertDialog` 显示保存成功提示
- 使用 `Scroll` 组件实现页面滚动

### 2. 状态管理
- 使用 `@State` 装饰器管理页面状态
- 使用全局变量 `globalSettings` 持久化设置
- 在 `aboutToAppear` 生命周期中初始化设置

### 3. 样式设计
- 遵循HarmonyOS设计规范
- 使用系统颜色资源（`$r('sys.color.xxx')`）
- 统一的卡片式布局风格
- 圆角和间距符合设计规范

## 功能演示流程

1. **进入设置页面**
   - 从主页点击"支付设置"按钮
   - 进入设置页面查看各项设置

2. **修改默认支付方式**
   - 点击"默认支付方式"项
   - 底部弹窗显示可选支付方式
   - 选择新的默认支付方式

3. **修改支付确认方式**
   - 点击"支付确认方式"项
   - 底部弹窗显示可选确认方式
   - 选择新的确认方式

4. **开关设置**
   - 切换"小额免密支付"开关
   - 切换"支付提醒"开关
   - 切换"自动保存银行卡"开关
   - 切换"显示交易详情"开关

5. **保存设置**
   - 点击"保存设置"按钮
   - 显示保存成功提示
   - 设置被持久化
